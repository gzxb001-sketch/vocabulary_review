# 前端 UI 设计最佳实践 Skill

本 Skill 综合了 **Refactoring UI** (Adam Wathan & Steve Schoger)、**Atomic Design** (Brad Frost)、**WCAG 2.2** 标准、以及 **Feature-Sliced Design** 等业界权威来源的核心经验。

## 核心理念

**构建设计系统，而非页面**。创建可复用、可扩展的组件来保证整个应用的一致性。

---

## 1. 配色系统

### 1.1 调色板原则
- 主色 (Primary): 定义品牌核心身份，占 UI 约 60%
- 辅助色 (Secondary): 区分板块信息，占 UI 约 30%
- 强调色 (Accent): 仅用于关键操作（如 CTA 按钮、删除），占 UI 约 10%
- 中性色: 背景、文字、边框用不同灰阶 (slate/gray/zinc)

### 1.2 颜色功能语义
| 颜色 | 情绪 | 典型用途 |
|------|------|----------|
| 蓝/靛蓝 | 信任、冷静 | 主按钮、导航、品牌 |
| 绿色 | 确认、成功 | 成功状态、完成提示 |
| 黄/橙 | 警告、注意 | 模糊状态、警告提示 |
| 红色 | 危险、紧急 | 错误、删除、不会 |

### 1.3 对比度要求 (WCAG AA)
- 正文 vs 背景: ≥ 4.5:1
- 大字 (≥18px bold 或 ≥24px): ≥ 3:1
- 用工具验证: WebAIM Contrast Checker

### 1.4 深色模式
- 背景用深灰 (#121212 或 #1E1E1E)，不用纯黑
- 白色文字亮度值控制在 20 以内
- 通过 CSS Variables 统一管理，过渡需平滑（60fps+）

## 2. 排版系统

### 2.1 字体层级
- H1: 26-32px, weight 700-800, letter-spacing -0.02em
- H2: 18-22px, weight 600-700
- Body: 15-16px, weight 400, line-height 1.5-1.7
- Caption/Label: 12-13px, weight 500-600, uppercase + letter-spacing

### 2.2 行高规则
- 正文 line-height: 1.5-1.75
- 标题 line-height: 1.2-1.3
- 行高应为字号的精确倍数（如 16px 字号 → 24px 行高）

### 2.3 字体选择
- 优先使用系统字体栈: `"Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- 中文可用: `"PingFang SC", "Microsoft YaHei", "Noto Sans SC"`
- 避免使用超过 2 种字体族

## 3. 间距与尺寸系统

### 3.1 间距递增基准
采用 4px/8px 基准递增:
```
xs: 4px   sm: 8px   md: 12px   lg: 16px   xl: 20px   2xl: 24px   3xl: 32px
```

### 3.2 间距语义
- 元素内部 padding: 12-16px
- 卡片圆角: 12-20px
- 卡片间距 (gap): 10-14px
- 区块间距: 16-24px
- 页面 padding: 16-20px 水平, 20-32px 垂直

### 3.3 触摸目标
- 最小可点击区域: 44x44px (移动端) / 36x36px (桌面端)
- 按钮 padding: 12-14px 垂直, 14-16px 水平

---

## 4. 组件设计

### 4.1 按钮
- 主按钮: 渐变背景 + 阴影 + hover 微上移 (transform: translateY(-1px))
- 次按钮: 白色背景 + 边框 + hover 背景微变
- 危险按钮: 红色渐变背景
- hover 必须比默认状态视觉上"更重"
- disabled 状态: opacity 0.4-0.6, cursor not-allowed
- 按钮文字永远用语义描述（"保存到词库"，而不是"确定"）

### 4.2 卡片
- 白色背景 + 圆角 (14-20px) + 细微阴影
- 多层阴影: `0 1px 3px rgba(0,0,0,0.04), 0 4px 16px rgba(0,0,0,0.06)`
- 边框: 1px solid 极浅灰，增强层次感

### 4.3 输入框
- 默认: 1.5px 浅灰边框
- Focus: 边框变为主色 + 3px 主色低透明度外发光 (box-shadow)
- 过渡: 0.15s ease
- 圆角: 10-12px
- 预留足够的 padding: 11px 14px

### 4.4 列表项
- 卡片式可点击项用圆角边框，hover 变色
- 帮助文字、时间戳用 muted 色（#64748b）
- 每项上方留操作区（多选、详情入口）

## 5. 交互状态

### 5.1 必须覆盖的状态
每个交互元素至少要有:
- **Default**: 正常态
- **Hover**: 悬浮态（桌面端）
- **Active/Pressed**: 按下态
- **Focus**: 键盘聚焦态（可见的 outline 或 ring）
- **Disabled**: 禁用态
- **Loading**: 加载态（按钮文字变"处理中..."）

### 5.2 微动效原则
- 过渡时间: 0.1-0.3s
- 使用 ease 或 ease-out 缓动
- 页面切换可加 fadeIn 动画（opacity + translateY(4px → 0)）
- 不要让动效超过 300ms，否则用户感觉"慢"

---

## 6. 视觉层次

### 6.1 信息层次
1. Primary: 标题/大数字 — 最大字号 + 最深颜色 (#0f172a)
2. Secondary: 副标题/释义 — 中等字号 + 中等深度 (#334155)
3. Tertiary: 提示/标签/时间 — 小字号 + 浅色 (#64748b, #94a3b8)

### 6.2 统计卡片色彩区分
用不同浅色背景区分统计指标:
- 总词条: 浅蓝 (#eff6ff)
- 今日新增: 浅绿 (#ecfdf5)
- 待复习: 浅黄 (#fef3c7)
- 已复习: 浅紫 (#f5f3ff)

---

## 7. 响应式设计

- 移动优先: 默认样式面向小屏，用 min-width 断点扩展
- 断点参考: sm: 640px, md: 768px, lg: 1024px
- 内容区最大宽度: 480px (移动 App 风格) 或 640-768px
- 图片/容器用 max-width: 100%

---

## 8. 可访问性 (A11y)

- 所有表单元素必须有 `<label>` 关联
- 图标按钮必须有 aria-label
- 颜色不能是传达信息的唯一方式
- 键盘导航: Tab 顺序合理，Enter/Space 可激活按钮
- focus 态要有可见指示器（浏览器默认可，但建议自定义 ring）

---

## 9. 空状态与错误状态

- 空状态: 居中图标 + 友好说明文字 + 引导操作按钮
- 加载状态: 简洁的"加载中..."文字或骨架屏
- 错误提示: 明确告知出了什么问题、用户可以做什么
- 成功反馈: 操作完成后的跳转或短暂提示

---

## 10. 导航设计

- 顶部导航: 品牌名 + 核心入口（2-3个）
- Sticky 定位，避免滚动时丢失
- 当前页面入口要有视觉区分（如 opacity 变化或不显示为链接）

---

## 快速自查清单

设计/评审界面时逐项检查:
- [ ] 是否有明确的主色、辅助色、强调色？
- [ ] 文字对比度是否达到 4.5:1？
- [ ] 触摸目标是否 ≥ 44px？
- [ ] 每个按钮是否有 hover/active/disabled/loading 四个状态？
- [ ] 间距是否基于 4px/8px 系统？
- [ ] 空状态和错误状态是否有友好提示？
- [ ] 字体层级是否不超过 3 级？
- [ ] 是否有顶部导航且 sticky？
- [ ] 表单元素是否有 focus 光环？
- [ ] 动效是否 ≤ 300ms？
